Inel Work Scheduler

A Windows desktop scheduler for individual streamers collaborating with video editors and thumbnailers.

What is this app?

Inel Work Scheduler is a Windows desktop application that helps an individual live streamer manage video production schedules together with their freelance video editors and thumbnail designers.

All work data is synchronized bidirectionally with a single Google Sheet that the user themselves owns. The app does not run any developer-side backend; it only uses the user's Google account permissions to access the user's own sheet.

Main features

Schedule management Shorts / Longform / Replay tabs with monthly paging and To-do / Done grouping.
Editor collaboration Per-editor / per-thumbnailer custom installers with their own permissions. Each user sees only their assigned rows.
Broadcast auto-logging Detects Chzzk live broadcasts and accumulates category / title timeline into the Replay tab.
Google Sheet sync Bidirectional sync with a sheet the user owns. The sheet is the single source of truth for all data.

Download & access

The app is currently in a private beta and distributed to a small number of individual users only. Please contact us via the email below if you would like to try it.

Runs on Windows 10 / 11 (64-bit).

OAuth permissions (Google sign-in)

The app requests the following Google OAuth scopes from the user. All scopes are used only within the boundary of the sheet that the user themselves owns.

  • Google Sheets access (spreadsheets) — Bidirectional sync of the schedule data inside the sheet whose URL the user explicitly entered. The app uses the user's sheet as its only data store, instead of maintaining its own database.
  • Google Drive (per-file restricted) (drive.file) — Used only for the metadata operations (e.g. creating per-year sub-sheets) on the sheet the user registered. The app does not enumerate or read any other Drive file.
  • Email lookup (userinfo.email) — Used to identify the signed-in account and to verify that an editor's Gmail matches the one their administrator registered for them.

The app does not transmit data to any external server, has no developer-side backend, no analytics, and no advertising. See the privacy policy for details.
