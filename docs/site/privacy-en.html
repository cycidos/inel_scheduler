Privacy Policy

Last updated: 2026-05-25

1Scope

This policy applies to all users of Inel Work Scheduler (the "App"). The App is a Windows desktop application developed and distributed by cycidos (the "Developer") for the purpose of personal live-streaming collaboration.

2Information we access

The App accesses the following information that the user either enters directly or grants access to via Google account permissions.

  • The Google account email address — Used to identify the signed-in user inside the App. Retrieved via the Google OAuth userinfo.email scope.
  • Content of the Google Sheet the user designated — The App reads and writes the data inside the sheet whose URL the user entered (video titles, work statuses, assigned editors, broadcast logs, etc.). Uses Google OAuth spreadsheets and drive.file scopes.
  • Public Chzzk broadcast information — Live status, title, and category of the user's own Chzzk channel. Retrieved via the public Chzzk API without authentication. This is not personal data.
  • Work data the user enters directly inside the App — Editor roster, work status options, category seed list, schedules, etc. All entered by the user through the App's UI.

The App does not collect user interaction logs, analytics, tracking, advertising IDs, location data, device fingerprints, or contacts.

3How we use the information

The information we access is used only for the following purposes.

  • Google Sheet synchronization — The data the user enters in the App is saved to the user's own Google Sheet, and changes in the sheet are reflected back into the App. The user's sheet is effectively the App's single data store.
  • User identification — For editor / thumbnailer builds of the App which have a specific email embedded at build time, the App verifies that the signed-in account matches the embedded email and prevents anyone else from using that build.
  • Broadcast auto-logging — The App polls Chzzk's public API for the user's own channel and writes broadcast events to the user's Google Sheet ("Replay" tab) automatically.

The information is not used for any purpose other than the above and is not transmitted to any third party other than the App's developer (and even the developer does not receive it — see section 4).

4Where the data is stored

The Developer does not operate any backend server, cloud database, analytics service, or advertising network. All data is stored only in the following two locations.

  • The user's own Google Sheet — The sheet whose URL the user entered in the App. Owned by the user and lives in the user's own Google Drive.
  • The user's local PC — The Windows %APPDATA%\Inel Work Scheduler\ folder and Electron's localStorage. The OAuth refresh token (encrypted by Electron's safeStorage using the OS keychain / Windows DPAPI), user settings cache, and user-added categories are stored here.

The App does not transmit the user's Google account, sheet content, or local data to any external location.

5Third-party sharing

The App does not share any user information with any third party.

Exception — when the user actively uses the App's "build editor/thumbnailer installer" feature to grant their own collaborators access to their sheet, those collaborators can read and edit the user's sheet. This is an explicit user action, not something the App performs automatically.

6OAuth scope usage

The Google OAuth 2.0 scopes the App requests and their usage are as follows. Each scope is used only after the user explicitly grants it.

  • https://www.googleapis.com/auth/spreadsheets — Used to bidirectionally synchronize the content of the spreadsheet whose URL the user provided. The App accesses only the single sheet the user told it about.
  • https://www.googleapis.com/auth/drive.file — A per-file restricted scope that grants access only to files the App itself created or to which the user explicitly granted access. The App does not enumerate the user's entire Drive.
  • https://www.googleapis.com/auth/userinfo.email — Used to look up the signed-in user's email for identification. The App does not retrieve any other profile information (name, photo, gender, etc.).

The App complies with the Google API Services User Data Policy, including the Limited Use requirements.

7Data retention & deletion

The user can delete all data the App stored on the user's local PC (encrypted OAuth refresh token, settings cache, etc.) by uninstalling the App.

  • From inside the App: Settings → Other Settings → "Delete App". Or via the Windows Control Panel, uninstall the App. Either way removes all locally stored data.
  • Revoke OAuth access: From Google Account → Security → Third-party apps, revoke "Inel Work Scheduler". After revocation, the App can no longer access the user's sheet.
  • Sheet data — The data the user entered through the App remains in the user's own Google Sheet. The user can delete the sheet or clear its contents directly.

8Security

The OAuth refresh token is encrypted via Electron's safeStorage API, which uses the OS keychain / Windows DPAPI underneath. Another OS user looking at the disk directly cannot read the token in plaintext.

All communication between the App and Google's servers uses HTTPS.

9Children's data

The App is not directed at children under the age of 14 and does not knowingly collect personal data of children.
