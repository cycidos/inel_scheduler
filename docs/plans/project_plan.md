# inel_scheduler 프로젝트 계획서 (요구사항 v0.2)

> 대상 스트리머: **이늘**
>
> 본 문서는 현재 논의된 요구사항을 기준으로 한 상세 초안이다.
> Google Sheets 헤드(컬럼)는 운영 중 변경될 수 있으며, 변경 가능성을 전제로 설계한다.
>
> 현재 협업 스케줄러 앱 단독 개발의 최신 확정안은 `docs/plans/work_scheduler_app_plan_v1.md`를 우선 기준으로 사용한다.

## 1) 프로젝트 목표
- 이늘과 편집자가 함께 사용하는 `업무 협업 스케줄러`를 만든다.
- 시청자와 공유하는 `방송 스케줄러`(월간/주간)를 만든다.
- 데이터 원천은 Google Sheets로 두고, 가능한 범위에서 치지직 API 데이터를 누적해 정확도를 높인다.

## 2) 배경 및 현재 운영 방식
- 현재 업무 협업은 Monday 기반으로 운영 중이다.
- 프로젝트 단위는 크게 3개:
  - 유튜브 숏폼
  - 유튜브 롱폼
  - 다시보기 영상
- 이번 프로젝트는 위 운영 방식을 Google Sheets + 자체 스케줄러로 옮기는 것이 1차 목표다.

## 3) 사용자 및 권한

### A. 관리자 (이늘)
- 업무 협업 스케줄러: 전체 조회/작성/수정/상태 변경
- 방송 스케줄러: 전체 조회/작성/수정/삭제/공개 배포
- 배포 권한: 시청자용 스냅샷 생성 및 배포 트리거

### B. 편집자
- 업무 협업 스케줄러: 조회 + 상태 변경 + 링크/메모 갱신
- 방송 스케줄러: 기본적으로 읽기 전용 (정책 확정 전)

### C. 시청자
- 방송 스케줄러 공개본 읽기 전용
- 직접 편집 불가

## 4) 업무 협업 스케줄러 요구사항 (Work Scheduler)

### 4-1. 목적
- 편집 업무의 진행 상태를 이늘/편집자가 함께 추적한다.
- 작업 지시, 원본 공유, 납품 링크까지 한 흐름에서 관리한다.

### 4-2. 트랙 구조 (고정)
- `Shorts`
- `Longform`
- `Full Replay`

### 4-3. Google Sheets 헤드(Shorts/Longform, 현재안)
- 방송날짜
- 영상제목
- 영상 카테고리
- 원본 공유
- 작업상태
- 편집시작일
- 납품일
- 납품 공유

#### 4-3-a. Shorts/Longform 상세 규칙
- `영상 카테고리` 선택값: `게임`, `노래음악`, `토크`
- 카테고리가 `게임`일 때만 게임명 입력 필드를 추가 노출한다.
- `작업상태` 선택값: `Wait`, `Wip`, `Done`, `Publish`, `Retake`, `Omit`
- `납품 공유` 링크는 `작업상태 = Publish`일 때만 활성화한다.
- `원본 공유`, `작업상태`, `납품 공유` 헤더에는 괄호 설명을 붙이지 않는다.

### 4-4. Google Sheets 헤드(Full Replay, 현재안)
- 방송날짜
- 영상제목
- 영상 카테고리
- 카테고리 타임라인
  - 라이브 방송 중 카테고리 변경 시점마다 기록
  - 방송 업타임 기준 타임라인으로 저장

#### 4-4-a. Full Replay 상세 규칙
- 헤더 표기는 괄호 설명 없이 유지한다.
- `영상 카테고리` 규칙은 Shorts/Longform과 동일하다.
  - 선택값: `게임`, `노래음악`, `토크`
  - `게임` 선택 시 게임명 입력 가능

### 4-5. 상태 정책 (초안)
- Wait: 대기
- Wip: 작업 중
- Done: 작업 완료
- Publish: 게시 완료
- Retake: 재작업 필요
- Omit: 제외/취소

### 4-6. 협업 정책
- 이늘과 편집자가 동일 화면/동일 데이터를 보고 상태를 갱신한다.
- 상태 변경, 링크 변경, 납품 관련 변경은 이력 추적이 가능해야 한다.

## 5) 방송 스케줄러 요구사항 (Broadcast Scheduler)

### 5-1. 목적
- 시청자에게 방송 예정 일정을 명확하게 공유한다.
- 관리자는 월간/주간 관점에서 빠르게 작성/수정한다.

### 5-2. 뷰/토글
- 월간 뷰 / 주간 뷰 토글
- 월간 표시: 방송 시작시간 + 제목 중심
- 주간 표시: 상세 내용까지 표시
- 방송 스케줄 표기 기준 연도는 `2026`으로 맞춘다. (GUI 샘플/초기 운영 기준)
- `+ Add Event` 버튼 없이 달력 날짜 클릭으로 추가/수정 모달을 연다.
- 휴무 설정된 일정은 월간/주간 모두 회색으로 표시한다.

### 5-3. 관리자 입력 필드
- 날짜
- 방송시간
- 제목
- 상세 내용
- 휴방 설정

### 5-4. 공개 범위
- 시청자는 공개된 데이터만 조회 가능
- 내부 메모/관리자 전용 정보는 비공개

## 6) 플랫폼/배포 시나리오

### 6-1. 웹 선택 시 (우선 고려안)
- 인프라: GitHub + Cloudflare
- 시청자 페이지는 스냅샷 기반으로 제공 (`inel_songbook` 방식 참고)
- 관리자 저장/배포 시 흐름:
  1) 관리자가 데이터 저장
  2) 스냅샷 생성 트리거
  3) GitHub Actions 빌드
  4) 스냅샷 배포
  5) 시청자 공개 반영
- 확장안: 네이버 카페 게시글로 자동/반자동 배포

### 6-2. 앱 선택 시
- 작성/수정 중심으로 운영
- 네이버 카페 게시글 배포까지 연결
- 시청자 공개 페이지 운영 방식은 별도 결정

## 7) 데이터 소스 및 동기화

### 7-1. 시트 분리 원칙
- 업무 협업 스케줄러와 방송 스케줄러는 **서로 다른 Google Sheets**를 사용한다.

### 7-2. 동기화 요구사항
- 앱/웹에서 수정한 내용은 시트에 반영되어야 한다.
- 시트에서 직접 수정한 내용도 서비스에 반영되어야 한다.
- 즉, 단방향이 아닌 실질적 양방향 반영 구조가 필요하다.

### 7-3. 동기화 설계 포인트 (추후 확정)
- 폴링 주기 (예: 1~5분)
- 충돌 처리 기준 (최근 수정 우선/필드 단위 병합)
- 실패 재시도 및 로그 적재 방식

## 8) 치지직 API 연동 방향
- 목적: 라이브 중 데이터(예: 시작/종료 시각, 상태, 카테고리 변화 단서) 수집
- 활용:
  - 방송 기록 정확도 보강
  - 다시보기 카테고리 타임라인 보조 데이터 생성
- 제약:
  - 인증 방식, 호출 제한, 수집 가능 필드 확인 필요

## 10) 비기능 요구사항
- 헤드(컬럼) 변경 가능성을 고려해 매핑 기반으로 구현
- 시청자용 페이지는 읽기 최적화/빠른 로딩 지향
- 관리자 작업은 저장 실패/동기화 실패 시 즉시 안내 필요
- UI는 기존 버전의 디자인 톤을 유지하고 기능만 확장한다.

## 11) 단계별 실행 계획
1. 요구사항 문서 고정 (`project_plan`, `code_flow`, rule)
2. Google Sheets 2종 스키마 확정 (업무용/방송용)
3. 동기화 레이어 설계 (시트 -> 앱, 앱 -> 시트)
4. 업무 협업 스케줄러 MVP 구현
5. 방송 스케줄러 MVP 구현 (월간/주간 토글)
6. 웹 경로 선택 시 스냅샷 배포 파이프라인 구축
7. 치지직 API 수집/누적 및 타임라인 보강
8. 네이버 카페 게시글 배포 흐름 연결

## 12) 결정 필요 항목 (Open)

- 편집자 권한 세부 범위 (어디까지 수정 가능한지)
- 스냅샷 갱신 방식 (수동/자동/혼합)
- 네이버 카페 배포 자동화 수준
- 치지직 API 호출/보관 정책
- 외부 연동 조사 문서: `docs/plans/integration_research_2026-04-08.md`

## 13) GUI 초안 문서 (Electron 기준)

### 13-1. 내비게이션 구조
- 상위 메뉴는 2개만 제공:
  - `Work Scheduler`
  - `Broadcast Scheduler`
- `Work Scheduler` 하위 메뉴:
  - `Shorts`
  - `Longform`
  - `Full Replay`

### 13-2. 전체 레이아웃
- 좌측: 메뉴/서브메뉴 트리
- 상단: 현재 뷰 타이틀, 공통 필터, 동기화 상태 배지
- 중앙: 메인 데이터 영역
- 우측(옵션): 선택 행 상세 패널(링크/코멘트/이력)

### 13-3. Work Scheduler 화면 초안 (ShotGrid 스타일)
- 카드형보다 그리드(테이블) 중심으로 구성한다.
- 행 단위 인라인 편집을 기본으로 한다.
- 상단 필터:
  - 상태
  - 카테고리
  - 편집자
  - 납품일 범위
- 핵심 동작:
  - 상태 즉시 변경
  - 링크 입력/열기
  - `Publish` 전에는 납품 공유 링크 비활성
  - 변경 이력 패널 확인

### 13-4. Broadcast Scheduler 화면 초안
- 월간/주간 토글 제공
- 이벤트 추가/수정 방식:
  - 달력의 날짜 셀 클릭 -> 편집 모달 오픈
- 입력 필드:
  - 날짜, 방송시간, 제목, 상세 내용, 휴방 설정
- 표시 규칙:
  - 월간: 시간 + 제목
  - 주간: 상세 내용 포함
  - 휴방: 회색 톤 카드/텍스트로 고정 표기

### 13-5. 테마/디자인 가이드
- 기본 톤: White + Soft Pink
- 대표 색상:
  - 배경: `#FFFFFF`, `#F9FAFB`, `#F3F4F6`, `#FDF2F8`
  - 포인트: `#EC4899`, `#EF7BA2`, `#FBCFE8`
  - 텍스트: `#1F2937`, `#374151`, `#6B7280`, `#9CA3AF`
- 업무 화면은 Maya ShotGrid 계열의 생산관리형 밀도(실용 우선)를 유지한다.
