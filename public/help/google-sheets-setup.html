Google Sheets 연결 설정 가이드

이늘 스케줄러 · 관리자용 (1차 배포)

이 가이드로 무엇이 되나요?

Google Cloud에서 발급받은 Service Account의 JSON 키를 등록하고, 작업용 시트를 공유해 두면 앱이 시트를 자동으로 가져오고 변경 내용을 다시 올릴 수 있어요.

  • 네이버 / 구글 계정으로 로그인하지 않아도 됨
  • JSON 한 번만 등록하면 PC에 영구 저장
  • 편집자/썸네일러는 자기 시트만 공유받으면 됨
  • 해지는 시트 공유만 끊으면 즉시 차단

1

Google Cloud 프로젝트 만들기

Google Cloud Console에 접속해 새 프로젝트를 만듭니다. 프로젝트 이름은 자유 (예: inel-scheduler).

  • 이미 사용 중인 프로젝트가 있다면 그대로 써도 됩니다.
  • 비용은 발생하지 않습니다. (Google Sheets API 무료 할당량 내 사용)
↗ Google Cloud Console 열기

2

Google Sheets API 활성화

좌측 메뉴 API 및 서비스 → 라이브러리에서 Google Sheets API를 검색하고 사용 버튼을 눌러 활성화합니다.

  • 이 단계가 빠지면 인증은 통과해도 시트 호출이 권한 오류로 실패합니다.
  • 한 번만 활성화하면 됩니다.
↗ Sheets API 라이브러리 바로가기
GIF 안내가 아직 등록되지 않았습니다.

3

Service Account (서비스 계정) 만들기

API 및 서비스 → 사용자 인증 정보 → +사용자 인증 정보 만들기 → 서비스 계정 메뉴로 이동해 새 서비스 계정을 생성합니다.

  • 이름: 식별 가능한 아무거나 (예: inel-sheet-bot)
  • 역할 부여, 사용자 액세스 권한은 건너뛰기 가능 (Sheet 공유로 권한 부여)
GIF 안내가 아직 등록되지 않았습니다.
public/help/gifs/03-create-sa.gif
4

JSON 키 다운로드

생성한 서비스 계정을 클릭 → 키 탭 → 키 추가 → 새 키 만들기 → JSON 선택 후 다운로드합니다.

  • 다운로드된 .json 파일은 컴퓨터의 안전한 위치에 보관하세요.
  • 이 파일이 곧 Sheets 접근 자격증명입니다.
GIF 안내가 아직 등록되지 않았습니다.
public/help/gifs/04-download-json.gif
5

앱에 JSON 키 등록

이늘 스케줄러 앱 → 설정 → 구글 시트 연결 탭에서 JSON 파일을 드래그&드롭하거나 파일 선택 버튼으로 등록합니다.

  • 등록되면 자동으로 서비스 계정 이메일이 표시됩니다.
  • JSON 파일 경로는 PC 로컬에 저장되며, 시트 외부로 전송되지 않습니다.

Google Sheets에 권한 부여

대상 Google Sheets 문서를 열고 우측 상단 공유에서 복사한 이메일을 편집자 권한으로 추가합니다.

  • 편집자가 아닌 뷰어 권한이면 앱에서 시트 쓰기가 실패합니다.
  • 나중에 권한 회수를 원하면 시트 공유에서 이 이메일을 제거하면 즉시 차단됩니다.

8

연결 테스트

앱 설정의 연결 테스트 버튼을 눌러 시트가 정상으로 보이는지 확인합니다. 성공하면 시트 이름과 탭 개수가 표시됩니다.

  • 실패 시 메시지를 확인하세요. 403 Permission denied 라면 7단계 권한 공유를 다시 점검하세요.
  • 404 Not Found 라면 시트 URL을 확인하세요.
