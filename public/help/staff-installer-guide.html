편집자 / 썸네일러 인스톨러 만들기

이늘 스케줄러 · 관리자 전용 가이드

이 가이드는 무엇을 다루나요?

각 편집자/썸네일러에게 본인 권한이 자동 임베드된 .exe 인스톨러를 만들어 전달하는 흐름과, 권한을 회수/재발급/영구 차단하는 운영 절차를 정리합니다.

받은 사람은 .exe 더블클릭 설치만 하면 별도의 시트 URL 입력 / 인증 키 등록 없이 본인 권한으로 바로 작업할 수 있어요.

  • 관리자 본인의 SA JSON 이 임베드됨
  • 토큰은 _tokens 시트에 자동 등록
  • 편집자 앱은 자기 토큰만 검증해서 잠금 해제
  • 10분 주기로 권한 재확인

1

사전 조건 (이 단계가 안 되어 있으면 빌드 버튼이 비활성)

빌드를 시작하기 전에 관리자 앱에 다음 4가지가 모두 준비되어 있어야 합니다.

  1. 설정 → 구글 시트 탭에서 Sheet URL 입력
  2. 같은 탭에서 Service Account JSON 드래그 등록 + [연결 테스트] 통과
  3. 구글시트 다운로드 1회 (시트 동기화 시작)
  4. 설정 → 시트 설정에서 해당 편집자/썸네일러 이름 + 역할 등록
SA JSON 이 새로 필요한 게 아니에요 — 처음 시트 등록할 때 만든 그 JSON 파일을 그대로 [구글 시트] 탭에 드래그 등록하면 됩니다. 관리자 본인의 SA 가 모든 편집자 인스톨러에 그대로 임베드됩니다.
SA JSON 자체를 처음 만드는 방법은 별도 가이드 [설정 → 구글 시트 → 설정 방법 자세히 보기] 에서 8단계로 설명되어 있어요.
2

빌드 다이얼로그 — 인스톨러 생성

  1. 설정 ⚙ → 기타 설정 탭으로 이동
  2. "편집자/썸네일러 전용 설치 파일 만들기" 카드에서 해당 인원 옆 [인스톨러 빌드] 클릭
  3. 모달이 열리면:
    • 출력 폴더 선택 — 예: release/editors/홍길동/ 또는 바탕화면
    • 연결 시트는 현재 등록된 URL 이 자동 표시 (수정 불가)
    • [빌드 시작] 클릭 → 약 30~40초 소요
  4. 완료 시 결과 카드 표시 + 출력 폴더에 다음 2개 파일 생성:
    • Inel Scheduler-Editor-Setup-1.1.0.exe (또는 -Thumbnailer-) — 본체
    • ...-Setup-1.1.0.exe.blockmap — 차분 업데이트용 메타 (같이 전달해도 됨)
  5. 그 .exe 파일만 편집자에게 전달 (USB / 메신저 / 클라우드 등)
같은 사람을 다시 빌드하면 토큰이 자동 갱신(rotate)되고 새 .exe 가 생성됩니다. 옛 .exe 는 자동으로 잠금 화면이 떠서 무효화됩니다.
3

_tokens 시트 — 자동 생성되는 권한 명단

첫 빌드 시 시트 안에 _tokens 라는 시트가 자동으로 만들어집니다. 관리자가 직접 만들 필요 없어요. 헤더는 다음과 같습니다.

name role token issuedAt status lastSeen
홍길동 editor 5f8e…hex 2026-05-25T01:30:00Z active 2026-05-25T02:15:00Z
김썸네일 thumbnailer 91a4…hex 2026-05-25T01:32:00Z active (편집자 실행 후 자동 채워짐)
  • token: 빌드 시 생성된 48자 hex. .exe 안에 임베드되어 시트와 대조.
  • status: active 면 통과, 그 외엔 잠금. 회수 시 이 값을 변경.
  • lastSeen: 편집자 앱이 마지막으로 검증한 시각 (자동 갱신). 누가 활동 중인지 추적용.
이 시트를 사람이 직접 편집해도 됩니다. 권한 회수 / 영구 차단은 모두 이 시트를 통해 이뤄져요 (자세한 절차는 §5 참조).
4

편집자가 받은 .exe 실행 시

편집자는 다음 흐름으로 사용합니다 — 관리자 안내 없이도 자기 권한으로 자동 동작.

  1. 관리자가 보낸 Inel Scheduler-Editor-Setup-1.1.0.exe 더블클릭
  2. NSIS 인스톨러 진행 (기본 경로: 내 문서\Inel Scheduler Editor)
  3. 설치 완료 후 앱 실행 → "권한 확인 중..." 잠깐 표시 → 본문 화면
  4. 본문에서 편집자가 볼 수 있는 것:
    • 본인이 담당자로 지정된 행
    • 상단바: [일정 새로고침] / [변경사항 저장] (구글 시트 노출 없음)
    • 설정 → [기타 설정] 탭 한 개만 노출 (자동실행 / 데이터 폴더 / 앱 삭제)
  5. 매 10분마다 자동으로 권한 재검증. 권한이 회수되면 잠금 화면으로 전환.
편집자에게 시트 URL / SA JSON 을 알려주거나 등록을 요청할 필요가 전혀 없습니다. 그 정보는 모두 .exe 안에 임베드되어 있어요.
5

권한 회수 / 재발급 / 영구 차단

운영 강도에 따라 4단계의 옵션이 있습니다. 작은 일에는 가벼운 옵션, 보안 사고 의심에는 강한 옵션을 씁니다.

① 일시 회수 매우 쉬움 · 영향 그 사람만

휴직, 잠깐의 정지, 외주 일시 중단 같은 상황.

  1. 구글 시트 → _tokens 시트 열기
  2. 해당 편집자 행의 status 컬럼을 active → revoked 로 수정
  3. 편집자 앱이 10분 주기 재검증 시점 (또는 재시작 시) 자동 잠금 화면 표시
  4. 다시 복귀시키려면 revoked → active 로 되돌리기. 다음 재검증 때 자동 해제.

② 재발급 (토큰 갱신 / rotation) 쉬움 · 그 사람만

토큰이 노출됐을 의심, 편집자 PC 분실, 정기 회전.

  1. 설정 → 기타 설정 → 같은 편집자 옆 [인스톨러 빌드] 다시 클릭
  2. 새 토큰이 발급되어 시트의 같은 행을 update (옛 토큰 무효화)
  3. 새 .exe 가 출력 폴더에 생성됨 → 편집자에게 전달
  4. 옛 .exe 는 자동으로 잠금 화면 (토큰 불일치)

③ 영구 차단 — 가벼운 방법 중간 · 그 사람만 (앱 차원)

퇴직, 외주 종료 등. 신뢰할 수 있는 편집자라면 이 정도로 충분.

  1. _tokens 시트에서 그 행을 삭제 또는 status 를 revoked 로 변경
  2. 편집자 앱이 잠금 화면으로 전환 → 사용 불가
한계: SA 키 자체는 살아있으므로, 외부 도구로 그 키를 직접 사용하면 시트 접근이 여전히 가능합니다. 신뢰하지 못하는 사람을 차단할 때는 ↓ 강한 방법 사용.

④ 영구 차단 — 강한 방법 (SA 키 자체 회전) 강함 · 모든 편집자 영향

키 노출 확실, 보안 사고 의심, 정기 회전 (연 1회 권장).

  1. Google Cloud Console → IAM 및 관리자 → 서비스 계정 ↗ 열기
  2. 해당 프로젝트 선택 → 사용 중인 SA (예: inel-scheduler-sa@…) 클릭
  3. 상단 [키] 탭 → 회전할 키의 우측 ⋮ 메뉴 → [키 삭제]
  4. 같은 화면에서 [키 추가] → JSON 키 새로 생성 → 다운로드
  5. 관리자 앱 [구글 시트] 탭으로 돌아와 새 JSON 드래그 등록 → [연결 테스트] 통과 확인
  6. 모든 편집자에게 새 인스톨러 재배포 (옛 .exe 의 SA 키는 죽음 → 새 키로 다시 빌드)
현재 운영은 모든 편집자가 같은 SA 를 공유하는 단순 모델이라, SA 키를 갈면 모든 편집자가 영향을 받습니다. 한 명만 영구 차단하면서 다른 편집자에게 영향을 안 주려면 ↓ 별도 SA 운영 방식이 필요.

(특수) SA 자체 삭제 (계정 폐기)

그 SA 를 다른 시트에서도 안 쓸 때만 사용. 키 회전보다 더 강함.

  1. 위 화면에서 SA 행의 ⋮ → [관리] → [서비스 계정 삭제]
  2. 그 SA 의 모든 키 영구 무효화 + 시트 공유 권한도 자동 해제
  3. 다른 시트에서도 그 SA 를 사용 중이면 모두 영향. 반드시 미리 확인.

(향후) 편집자별 별도 SA 운영

편집자마다 새 SA 를 만들고 그 키를 해당 인스톨러에만 임베드하면, 한 명 영구 차단 시 다른 편집자에게 영향이 없습니다. 절차는 처음 시트 등록 시 SA 만든 절차와 100% 동일 — Cloud Console 에서 같은 프로젝트에 새 SA 추가 + JSON 키 발급 + 시트에 그 SA 이메일을 편집자 권한으로 공유.

다만 현재 1.1.x 빌드 다이얼로그는 관리자 본인의 SA 만 자동 임베드하므로, 이 운영 방식은 향후 업데이트에서 모달에 "이 편집자용 SA JSON 첨부" 입력란이 추가되어야 적용 가능합니다.
6

흔한 에러 / 트러블슈팅

  • "Service Account JSON 이 없습니다" → §1 사전 조건 ②번 미완료. 설정 → 구글 시트 탭에서 SA JSON 드래그 등록 후 다시 시도.
  • "시트 URL 이 없습니다" → §1 사전 조건 ①번 미완료. 같은 탭에서 Sheet URL 입력 후 다시 시도.
  • "먼저 [시트 설정] 탭에서 편집자/썸네일러를 등록하세요" → §1 사전 조건 ④번 미완료. [시트 설정] 탭에서 이름 + 역할 추가.
  • 빌드 중 멈춤 → 디버그 패널 켜서 로그 확인. 보통 npm/electron-builder 의 환경 변수 문제이거나 release/ 폴더 권한 문제. 안티바이러스가 새 .exe 를 잠시 잠그는 경우도 있음.
  • 편집자 앱에서 잠금 화면 → _tokens 시트의 해당 행 status 확인. active 가 아니면 변경 후 편집자에게 앱 재시작 안내.
  • 편집자 앱 시작 직후 "권한 확인 중" 무한 대기 → 인터넷 연결 불안정 또는 임베드된 시트 URL 이 잘못된 경우. 새 인스톨러 재발급 권장.
  • 같은 PC 에 관리자/편집자 둘 다 설치 → 가능합니다 (appId 분리). 시작메뉴에서 두 개 따로 보임 ("Inel Work Scheduler" / "이늘 스케쥴러 (편집자)").
이 문서는 앱 안에서 [인스톨러 빌드 모달 → 도움말 ↗] 버튼으로 언제든 다시 열 수 있습니다.
