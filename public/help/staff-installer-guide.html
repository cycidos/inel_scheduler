편집자 / 썸네일러 인스톨러 만들기

이늘 스케줄러 · 관리자 전용 가이드

이 가이드는 무엇을 다루나요?

각 편집자/썸네일러에게 본인 이메일과 시트 URL이 자동 임베드된 .exe 인스톨러를 만들어 전달하는 흐름과, 권한을 회수하는 방법을 정리합니다.

받은 사람은 .exe 더블클릭 설치 → 본인 Google 계정으로 로그인만 하면 바로 작업할 수 있어요.

  • 시트 URL + 스태프 이메일 자동 임베드
  • 각자 본인 Google 계정으로 직접 로그인
  • 권한 회수 = 시트 공유에서 해당 Gmail 제거 (1클릭)
  • 매 10분 주기로 권한 재확인

1

사전 조건 (빌드 버튼이 비활성이면 여기부터 확인)

빌드를 시작하기 전에 관리자 앱에 다음 3가지가 준비되어 있어야 합니다.

  1. 설정 → 구글 시트 탭에서 [Google 계정으로 로그인] 완료
  2. 같은 탭에서 Sheet URL 입력 (연결 테스트 통과 권장)
  3. 설정 → 시트 설정에서 해당 편집자/썸네일러를 이름 + Gmail 모두 입력해 등록
스태프의 Gmail 이 시트 공유 권한에도 추가되어 있어야 스태프 앱이 시트에 접근할 수 있습니다. 시트 우상단 [공유] 버튼에서 그 Gmail 을 편집자 권한으로 추가해두세요.
2

빌드 다이얼로그 — 인스톨러 생성

  1. 설정 ⚙ → 기타 설정 탭으로 이동
  2. "편집자/썸네일러 전용 설치 파일 만들기" 카드에서 해당 인원 옆 [인스톨러 빌드] 클릭
  3. 모달이 열리면:
    • 출력 폴더 선택 — 예: 바탕화면 또는 release/editors/홍길동/
    • 연결 시트는 현재 등록된 URL 이 자동 표시 (수정 불가)
    • [빌드 시작] 클릭 → 약 30~40초 소요
  4. 완료 시 결과 카드 표시 + 출력 폴더에 다음 2개 파일 생성:
    • Inel Scheduler-Editor-Setup-X.Y.Z.exe (또는 -Thumbnailer-) — 본체
    • ...-Setup-X.Y.Z.exe.blockmap — 차분 업데이트용 메타
  5. 그 .exe 파일만 편집자에게 전달 (USB / 메신저 / 클라우드 등)
임베드되는 정보: 시트 URL · 스태프 이름 · 스태프 Gmail · role (editor / thumbnailer). 인증 키나 토큰은 임베드되지 않습니다. 스태프는 본인 Google 계정으로 직접 로그인합니다.
3

편집자가 받은 .exe 실행 시

편집자는 다음 흐름으로 사용합니다 — 관리자 안내 없이도 자기 권한으로 자동 동작.

  1. 관리자가 보낸 Inel Scheduler-Editor-Setup-X.Y.Z.exe 더블클릭
  2. NSIS 인스톨러 진행 (기본 경로: 내 문서\Inel Scheduler Editor)
  3. 설치 완료 후 앱 실행 → 🔑 "Google 계정으로 로그인" 화면
  4. [Google 계정으로 로그인] 클릭 → 시스템 브라우저에서 Google 로그인 페이지 열림
  5. 관리자가 등록한 본인 Gmail로 로그인 + 권한 허용
  6. 앱으로 자동 복귀 → 본인 담당 행만 표시되는 본문 진입
  7. 매 10분 주기로 자동 권한 재확인. 시트 공유 권한이 해제되면 다음 새로고침 시 동기화 실패.
편집자에게 시트 URL / 인증 키를 알려줄 필요가 없습니다. 시트 URL 은 임베드, 인증은 본인 Google 계정으로 자동.
등록과 다른 Gmail 로 로그인하면 "계정 불일치" 잠금 화면이 표시되고 사용 불가. 관리자가 등록한 그 Gmail 이어야 합니다.
4

권한 회수 / 재발급

① 권한 회수 매우 쉬움 · 1클릭

휴직, 퇴직, 외주 종료 등 모든 상황에 동일하게 적용.

  1. 관리자가 작업 시트 우상단 [공유] 버튼 클릭
  2. 해당 편집자의 Gmail 옆 권한 메뉴 → [권한 제거] 또는 [액세스 권한 삭제]
  3. 그 편집자의 앱은 다음 [일정 새로고침] 시점에 시트 API 가 403 반환 → 동기화 실패
  4. 편집자가 .exe 를 보관하고 있어도 시트 접근 불가 → 사실상 완전 차단
Google 계정 기반이라 회수가 즉시 + 영구적입니다. 옛 SA 키 방식처럼 키 회전 / 모든 편집자 재배포 필요 없음.

② 재발급 (이메일 변경 / 정보 갱신) 쉬움

  1. 관리자 앱 [시트 설정] 탭에서 그 편집자를 [X] 로 제거
  2. 새 정보로 다시 등록 (이름 / 새 Gmail / role)
  3. 시트 [공유] 에서도 옛 Gmail 제거 + 새 Gmail 추가
  4. [기타 설정] → [인스톨러 빌드] 다시 → 새 .exe 전달

③ 임시 정지 (휴직 등)

완전 회수 대신 잠시 비활성화하려면 시트 [공유] 에서 그 Gmail 의 권한을 편집자 → 뷰어로 낮추세요. 앱은 시트를 읽기만 가능하고 [변경사항 저장] 시 403 받음. 복귀 시 다시 편집자 권한 복원.

5

흔한 에러 / 트러블슈팅

  • "Google 계정으로 먼저 로그인하세요" → 관리자가 §1 사전 조건 ①번 미완료. 관리자 앱에서 OAuth 로그인 후 다시 시도.
  • "시트 URL 이 없습니다" → §1 사전 조건 ②번 미완료. [구글 시트] 탭에서 URL 입력 후 다시 시도.
  • "이 스태프의 Gmail 미등록" → [시트 설정] 탭에서 해당 인원의 이메일 입력 후 다시 시도.
  • 편집자 앱 - "계정 불일치" 잠금 → 관리자가 등록한 Gmail 과 로그인 시도한 Gmail 이 다름. [로그아웃 후 다시 로그인] 으로 본인 Gmail 사용.
  • 편집자 앱 - 새로고침 시 권한 오류 (403) → 시트 공유에 그 Gmail 이 추가되어 있지 않음. 관리자가 [공유] 에 추가.
  • 편집자 앱 - "Google 계정으로 로그인" 안 됨 (Test mode 한정) → 개발자가 Cloud Console 의 Test users 명단에 그 Gmail 을 추가해야 함. 개발자에게 그 Gmail 전달.
  • 같은 PC 에 관리자/편집자 둘 다 설치 → 가능 (appId 분리). 시작메뉴에서 두 개 따로 보임.
  • 빌드 중 멈춤 → 안티바이러스가 새 .exe 를 잠시 잠그는 경우. 잠시 기다린 후 모달 닫고 재시도.
이 문서는 앱 안에서 [인스톨러 빌드 모달 → 도움말 ↗] 버튼으로 언제든 다시 열 수 있습니다.
